## Important Differences From The Documents

This is not the completed design. Treat it as a working scaffold plus Phase 1/3 prototype.

Not implemented yet:

- No new GitHub repo was created or pushed.
- No GitHub Packages or npm publish config.
- No CI workflow.
- No `CHANGELOG.md`.
- No browser/headless Chromium test run.
- No full engine contract fixture suite.
- No live env-gated smoke tests.
- No downstream adoption in the five portfolio repos.
- No `ai-agent-skills` skill update.

Implemented differently:

- `strict: true` is enabled, but I removed `exactOptionalPropertyTypes` after it made the initial scaffold noisy. Revisit if desired.
- Engines are practical first-pass implementations, not complete contract-grade provider engines.
- Agent argument validation is intentionally minimal and hand-rolled. It is not a full JSON Schema validator because the design asked for zero runtime dependencies.
- Anthropic JSON mode via forced tool-use is not fully implemented.
- Native tool-call streaming support is incomplete outside the OpenAI-compatible path.
- Ollama model capability probing via `/api/show` is not implemented.
- Google/Gemini function-call roundtrip behavior is not fully implemented.

## Cautions

- Do not add default model blocklists to the library. The design explicitly says governance is neutral by default.
- Do not silently set browser-dangerous provider headers. Apps must opt into those.
- Do not introduce SDK dependencies into the core unless the user changes the design goals.
- Do not store prompt/response content in handler telemetry. The handler records sizes; apps decide whether content storage is appropriate.
- Do not conflate model identity by bare model id. Preserve `(source, model)` provenance.
- Do not claim downstream adoption is complete until each repo's own tests and docs are updated.
