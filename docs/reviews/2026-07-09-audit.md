**Version audited:** `0.4.0` (branch base `6a5d4c9`, after PR #17's audit fixes)
**Scope:** full `src/` read (all 22 files), `tests/`, `examples/`, CI workflows, packaging, docs, and the prior audit (`AI Nugget Comprehensive Review & Au.md`) for follow-through.

---

## Executive summary

AI Nugget is a genuinely well-built library: a zero-dependency, isomorphic TypeScript core with a cleanly enforced pipeline (policy → keys → hooks → concurrency → retry → adapter → redacted telemetry), a data-driven provider table over four protocol engines, and a small honest agent loop. The prior audit's high-priority findings were actually fixed (verified below), the test suite is real (109 contract tests, same suite re-run in headless Chromium), and the documentation is unusually good for a project this size.

This audit found **one confirmed correctness bug** (a tool returning `undefined` kills an entire agent run), a handful of design-level robustness gaps (deterministic errors classified as retryable; a total-stream timeout that will kill long, healthy streams), and — the main theme — a set of **usefulness gaps** that push real applications back toward provider SDKs: no structured-output typing, no embeddings, no reasoning/thinking support, no prompt caching, no per-call provider escape hatch, and no cost accounting.

## Findings (new in this audit)

Ordered by severity. F1 is the only one verified as a hard failure; the rest are verified in source but not exercised live.

### F1 — CONFIRMED: a tool returning `undefined` kills the whole agent run

`src/agent/loop.ts:168` does `content: JSON.stringify(executed.result)`. `JSON.stringify(undefined)` returns `undefined` (it does not throw, so the surrounding `try` never fires), producing a `ChatMessage` whose `content` is `undefined` — a value the `ChatMessage` type forbids and no adapter defends against. On the next step, `openaiChat`'s message mapper calls `message.content.map(...)` and throws `TypeError`.

**Reproduced in this session:** a `defineTool` whose `execute` returns nothing (an extremely common shape — "fire the webhook, return nothing") ends the run with `stopReason: 'error'` after burning all retry attempts. Repro: mock a native tool-call round, execute a void tool, observe `stopReason: 'error'` at step 2 instead of a completed step 3.

**Fix (one line):**

```ts
// src/agent/loop.ts — tool result append
messages.push({ role: 'tool', toolCallId: call.id, name: call.name,
  content: JSON.stringify(executed.result) ?? 'null' });
```

Plus a regression test: "tool returning undefined feeds `null` back to the model and the run finishes."

### F2 — Deterministic request-build errors are retried as `network`

The engines build the request body *inside* the same `try` that wraps the wire call, and `streamError` → `fromUnknown` maps any non-`AIError` `Error` to `kind: 'network'` (retryable). So a deterministic failure — F1's `TypeError`, a circular `metadata` object, any body-serialization bug — is retried `maxAttempts` times with backoff, delaying the inevitable error by seconds and mislabeling it `network`. Suggested fix: build the body before entering the retried section, or classify `TypeError`/`RangeError` as `invalid_request`, `retryable: false` in `fromUnknown`.

### F3 — The stream timeout is a *total* deadline, not an idle timeout

`streamTimeout` arms one `withTimeout(conn.timeoutMs ?? 120_000)` covering the entire stream lifetime (deliberately, per the `postResponse` doc comment). Consequence: a healthy stream that takes longer than `timeoutMs` — a long completion on a slow local model, easily >120 s on Ollama with a large context — is killed mid-output with `kind: 'timeout'`, and because output was emitted, it is not retried. Callers can raise `timeoutMs`, but then a genuinely stalled stream hangs that long too. See I4 for the fix shape (separate `idleTimeoutMs` that resets per chunk).

### F4 — `AgentResult` doesn't say *what* failed

A run can end with `stopReason: 'error'` (or `'canceled'`, `'deadline'`), but `AgentResult` carries no error object. An app that consumes `runAgent(...).result` (rather than replaying the event stream) cannot distinguish auth failure from rate limit from F1. See I3.

### F5 — `promptJson` mode never tells the model the tools' parameter schemas

`withPromptJsonInstruction` (`src/agent/loop.ts:228`) lists only `name: description`. The model is asked to emit `{"tool":"name","input":{...}}` while having to *guess* what `input` should contain — then `validateToolArgs` rejects its guess and the loop spends steps on correction round-trips. Since `promptJson` is the default for every local runtime, this directly degrades the library's primary local-model use case:

```ts
// include the schema; it's already sitting in tool.parameters
`Available tools:\n${tools.map((t) =>
  `${t.name}: ${t.description}\n  input schema: ${JSON.stringify(t.parameters)}`).join('\n')}`
```

### F6 — Assistant messages with array content lose their parts when tool calls are present

In both `anthropic.ts:186` and `google.ts:161`, the tool-calling branch only forwards content when `typeof m.content === 'string'` — an assistant turn with `ContentPart[]` content plus `toolCalls` silently drops all its text/image parts on replay. Low frequency (models rarely produce this today), but it's a silent history-corruption path.

### F7 — Azure OpenAI `api-version` is frozen in the URL template

`profiles.ts:58` hardcodes `api-version=2024-10-21`. Azure retires API versions on a schedule; when this one goes, every consumer is stuck until a library release. Make it a quirk parameter (and overridable per connection), e.g. `quirks: { urlTemplate: ..., azureApiVersion: '2024-10-21' }` with `{apiVersion}` substitution.

### F8 — Minor / cosmetic

- **README claims "97 tests"; the suite is now 109** (README.md, Commands section). The kind of drift a doc-count-free phrasing would avoid.
- **`SessionRedactor.redact` rebuilds the entire redactor** (Set + closure) on every call (`redact.ts:44`); the handler redacts every telemetry record twice through it. Cache the built redactor and invalidate on `addSecret`.
- **`promptChars` counts base64 image bytes as prompt characters** (`util.ts:17`) — telemetry-inflating (a single image ≈ hundreds of KB of "prompt chars"). If intentional, name it; if not, weight or exclude image parts.
- **Ollama `listModels` probes `/api/show` serially per model** — correct-but-slow (prior audit noted it; still true). A concurrency of 3–4 would keep the "don't overwhelm local server" intent while cutting latency.

## Improvement ideas (nothing here is broken)

These are the concrete implementations I'd invest in, ranked by usefulness-per-effort. Each respects the stated non-goals (no routing, no policy defaults, zero runtime dependencies).

### I1 — Typed structured output via Standard Schema (biggest DX win, zero deps)

The README's JSON recipe hand-rolls `parseSprint` with `requireString`/`requireNumber`. The [Standard Schema](https://standardschema.dev) interface (`~standard`) is implemented by Zod, Valibot, and ArkType and costs the library **zero dependencies** — it's just a structural interface. One new function turns the whole recipe into a typed one-liner:

```ts
// src/json.ts
import type { StandardSchemaV1 } from './standard-schema.js'; // ~30-line vendored interface

export async function chatParsed<S extends StandardSchemaV1>(
  handler: AIHandler, conn: Connection, req: ChatRequest, schema: S,
): Promise<StandardSchemaV1.InferOutput<S>> {
  const result = await handler.chat(conn, {
    ...req,
    responseFormat: { type: 'json', schema: toJsonSchemaIfPossible(schema) },
  });
  const value = extractJson(result.text);
  const parsed = await schema['~standard'].validate(value);
  if (parsed.issues) throw new AIError(formatIssues(parsed.issues), { kind: 'invalid_response' });
  return parsed.value;
}

// app code becomes:
const sprint = await chatParsed(handler, conn, req, z.object({ headline: z.string(), minutes: z.number() }));
```

The same interface can back `defineTool` argument validation, replacing the "light validation only" caveat with real validation *when the app brings a validator*, while keeping today's light check as the no-validator default.

### I2 — Per-call provider escape hatch: `providerOptions`

Every provider capability the library doesn't model today (OpenAI `reasoning_effort`, Anthropic `thinking` and `cache_control`, Gemini `safetySettings`, Ollama `keep_alive`…) currently requires a library release. A single passthrough field removes that bottleneck without adding any policy:

```ts
// types.ts
export interface ChatRequest {
  // ...
  /** Merged into the provider request body last. You own compatibility. */
  providerOptions?: Record<string, unknown>;
}

// each engine's body builder, one line at the end:
return { ...body, ...req.providerOptions };
```

This is the difference between "the nugget can't do X yet" and "the nugget doesn't need to model X yet." Deep-merge for nested keys (`generationConfig` on Google) is the only subtlety worth a test each.

### I3 — Make agent failures inspectable (pairs with F1/F4)

```ts
export interface AgentResult {
  // ...
  /** Present when stopReason is 'error' — the terminal AIError. */
  error?: AIError;
}
```

Three-line change in `run()` (capture `streamFailure`/caught error into `makeResult`), plus the F1 fix and a `promptJson` schema line from F5, turns the agent layer from "demos well" to "debuggable in production."

### I4 — Idle timeout for streams (fixes F3 properly)

```ts
// transport.ts — withTimeout grows an optional idle reset
export function withTimeout(totalMs: number, external?: AbortSignal, idleMs?: number) {
  // arm two timers; expose bump() that re-arms the idle timer
  // textLines() calls timeout.bump() after every successful reader.read()
}
```

Default `idleTimeoutMs` to something aggressive (say 30 s between chunks) while leaving `timeoutMs` as the total backstop. A stalled provider is caught in seconds; a slow-but-alive local model is never killed mid-answer. This is the single highest-value robustness change for the local-runtime audience.

### I5 — Reasoning/thinking passthrough

Modern models (Claude extended thinking, DeepSeek R1, OpenAI o-series) emit reasoning the adapters currently drop on the floor (Anthropic `thinking_delta` falls through the delta-type checks; DeepSeek's `reasoning_content` is ignored by the OpenAI engine). The `StreamEvent` union already has the right slot — no contract change needed:

```ts
// anthropic.ts, in content_block_delta handling:
} else if (delta?.type === 'thinking_delta') {
  yield { type: 'context', kind: 'reasoning', data: { text: asString(delta.thinking) ?? '' } };
}
// openaiChat.ts, next to delta.content:
const reasoning = asString(delta?.reasoning_content);
if (reasoning) yield { type: 'context', kind: 'reasoning', data: { text: reasoning } };
```

Apps get live "model is thinking…" UI for free; requesting thinking stays app territory via I2's `providerOptions`.

### I6 — Cost accounting as a seam, not a table

Shipping a price table would rot instantly; a seam matches the library's philosophy (app owns policy, nugget owns plumbing):

```ts
export interface HandlerOptions {
  // ...
  /** App-supplied pricing; result lands on CallRecord.estimatedCostUsd. */
  cost?: (r: Pick<CallRecord, 'provider' | 'model' | 'usage'>) => number | undefined;
}
```

One optional field on `CallRecord`, one call site in `record()`. Combined with the existing `modelRef` identity, apps get per-model spend dashboards from telemetry alone.

### I7 — Embeddings (the largest scope decision)

Chat-only means any app adding retrieval outgrows the nugget immediately. Embeddings fit the existing architecture with a small surface — same pipeline, same key handling, same telemetry:

```ts
export interface EmbedRequest { model: string; input: string[]; metadata?: Record<string, unknown>; }
export interface EmbedResult { vectors: number[][]; usage: Usage; model: string; source: ModelSource; }
export interface ProviderAdapter {
  // ...
  embed?(conn: ResolvedConnection, req: EmbedRequest): Promise<EmbedResult>;
}
// handler.embed() wraps it in the standard pipeline like runProbe does today
```

`openaiChat` (`POST /embeddings`), `google` (`:batchEmbedContents`), and `ollama` (`/api/embed`) cover nearly every profile row. If this stays a non-goal, say so explicitly in the Non-goals section — right now it's just absent.

### I8 — Smaller items, in one pass

- **Provider rows:** `cohere` (compat mode), `perplexity`, `cerebras`, `moonshot` are each a profile-table row + fixture test via the existing `add-provider` skill. Cheap surface-area wins.
- **Parallel tool execution:** `budget`-style opt-in (`toolConcurrency?: number`) with `Promise.all` batches in the tool loop; sequential stays the default.
- **Prompt-caching usage fields:** add optional `cachedInputTokens` to `Usage` and read `cache_read_input_tokens` (Anthropic) / `prompt_tokens_details.cached_tokens` (OpenAI) — pure parsing, pairs with I6 for real cost numbers.
- **Dedupe engine helpers:** `randomId`, `safeParse`, `parseArgs`, `textContent` exist in 3–4 copies across `engines/*.ts`; move to `engines/base.ts`.
- **Ops:** add a lint job (Biome is the zero-config fit for a zero-dep project), a coverage gate (`@vitest/coverage-v8`, threshold ~85%), Dependabot for the dev tree, and `npm publish --provenance` in `publish.yml`.

## How to raise each score

- **Architecture & design (9.0 → 9.5):** Land I2 (`providerOptions`) and decide I7 (embeddings in, or an explicit non-goal). Both close the "future contract break" risks; nothing else structural is wrong.
- **Code quality (8.5 → 9.5):** Add Biome (lint job in CI), dedupe the 4× engine helpers into `base.ts`, fix the `SessionRedactor` rebuild (F8). All mechanical.
- **Correctness & robustness (7.5 → 9):** Fix F1 (one line + test), F2 (classify deterministic errors non-retryable), F3/I4 (idle timeout), F6 (array-content replay). Add an adversarial fixture suite: malformed SSE mid-stream, void/circular tool results, non-JSON bodies with JSON content-types.
- **Security & privacy (8.5 → 9+):** Add a secret-scanning CI job (gitleaks), extend redaction to URLs in error messages (a key pasted into `baseUrl` would currently survive `HTTP 404: …` errors unless it matches a token pattern), and document `literalKeySource` as test-only.
- **Testing (8.0 → 9):** Coverage gate in CI; agent-loop edge-case tests (F1's class of bug is exactly what a "tool returns undefined/null/circular/huge" quartet catches); a fuzz-ish malformed-provider-response suite. The live matrix already covers the hardest gap (real wire drift).
- **Documentation (9.0 → 9.5):** Remove volatile counts ("97 tests"), generate an API reference (typedoc → GitHub Pages) so the README stops being the only complete surface listing, add the one missing diagram (pipeline boxes).
- **Developer experience (7.5 → 9):** I1 (typed outputs) is most of it; I3 (inspectable agent errors) and F5 (schemas in promptJson) remove the two sharpest agent-layer edges.
- **Usefulness (7.0 → 9):** In order of leverage: I2 (unblocks every unmodeled provider feature immediately), I1 (typed structured output — the #1 friction in `DEVELOPMENT_REPORT.md`), I5 (reasoning models are the default now), I7 (embeddings decides whether apps *stay* on the nugget), I6 + caching fields (cost visibility). Each is independently shippable.
- **Operations (8.0 → 9):** Lint + coverage jobs, Dependabot, npm provenance, example lockfiles + bump mini-apps off `^0.3.1` once 0.4.x publishes.

## Conclusion

The nugget's fundamentals — pipeline enforcement, key hygiene, honest failures, isomorphism, docs — are better than most libraries many times its size, and the previous audit cycle demonstrably closed its findings. The work that matters now is different in kind: one real bug to fix in the agent loop (F1, with F2/F5 adjacent), one robustness design decision (idle vs. total stream timeout), and then a deliberate usefulness push (I1, I2, I5, and an embeddings decision) so that apps which start on AI Nugget don't have to leave it the month they need structured output types, reasoning traces, or retrieval.
